Cloud Run is a serverless platform for running containerised applications. A developer can package a web application into a container and deploy it without managing the underlying server infrastructure. The application can then be accessed through a URL, and Cloud Run handles the request and response process.
When a user visits a web application, the browser sends an HTTP request to the service. The service processes the request and returns an HTTP response, often containing HTML or structured data. This request-response model is important in the coursework because the text analysis application is triggered through a web interface.
Parallel processing improves efficiency when several files need to be analysed. Instead of processing files one after another, the application can process multiple files at the same time. This is useful for the multiple-file analysis section of the coursework. Cloud Run also supports scalable request handling, which means the deployed service can respond to changing demand more effectively than a single local program.
